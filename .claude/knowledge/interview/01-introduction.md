# 자기소개 및 경력 요약

## 1분 자기소개

> 안녕하세요, 7년차 백엔드 개발자 김대겸입니다.
>
> 대규모 데이터 파이프라인 설계와 안정적인 시스템 운영에 강점을 가지고 있습니다.
>
> 한화솔루션에서 50만대 IoT 장비의 실시간 데이터를 처리하는 Telemetry 시스템을 설계했고, Kafka 기반 Batch Consumer와 Bulk Insert로 분당 50만 건 처리를 달성했습니다. DLT 기반 재처리 메커니즘으로 데이터 정합성을 98%에서 100%로 개선한 경험이 있습니다.
>
> 현재 42dot에서 현대자동차 SDV 차량 데이터 플랫폼을 개발하고 있으며, Testcontainers 기반 테스트 환경 구축으로 커버리지 90%를 달성하고, OpenFGA 기반 권한 관리 시스템을 설계했습니다.
>
> [지원 회사]에서 제 경험을 바탕으로 안정적이고 확장 가능한 시스템을 만들고 싶습니다.

   - Spring Camp 발표, 블로그 운영
   - 팀 내 기술 세션 주도

### Q3: 본인의 약점은 무엇인가요?

**답변:**
- **약점:** 완벽주의 성향으로 일정 압박 시 스트레스

  - MVP 먼저, 점진적 개선 마인드셋
  - 코드 리뷰에서 "good enough" 기준 명확히 하기
  - 타임박싱으로 의사결정 속도 높이기

### Q4: 왜 이직하려고 하나요? / 퇴사 사유

**답변 (42dot → 타사):**

- 더 큰 규모의 트래픽/사용자를 경험하고 싶음

**주의:** 전 회사 비난 금지, 긍정적 이유 중심

### Q5: 왜 우리 회사에 지원했나요?

**답변 구조:**
1. 회사/제품에 대한 구체적 관심 포인트
2. 내 경험과의 연결고리
3. 기여할 수 있는 부분

### Q6: 5년 후 어떤 개발자가 되고 싶나요?

**답변:**
- 단기 (1-2년): 도메인 전문성 + 시스템 안정성 기여
- 중기 (3-5년): 기술 리드로서 아키텍처 의사결정 주도
- 장기: 팀의 기술 문화를 만들어가는 개발자

## 꼬리 질문 대비

### Q: 왜 Android에서 Backend로 전환했나요?

**답변:**
- 클라이언트보다 서버 로직에 더 흥미를 느낌
- 대규모 데이터 처리, 분산 시스템에 관심
- 우아한테크캠프 Pro를 통해 백엔드 역량 강화 후 전환

### Q: 한화솔루션에서 가장 어려웠던 점은?

**답변:**
- 레거시 시스템과의 공존
- 기존 수만 대 → 50만 대 스케일업 요구사항
- 해결: 점진적 마이그레이션, 부하테스트 기반 검증

### Q: 42dot에서 가장 임팩트 있었던 일은?

**답변:**
- Testcontainers 기반 테스트 환경 구축
- 팀 전체 테스트 문화 개선
- 배포 자신감 향상, 버그 사전 발견율 증가

## 기술 스택 질문 대비

### Q: 주력 언어/프레임워크는?

**답변:**
- **Kotlin + Spring Boot** (2년+)
- Java 경험 기반, Kotlin의 null safety, 간결함 선호
- JPA, QueryDSL, Kotest

### Q: 왜 Kotlin을 선호하나요?

**답변:**
1. Null Safety - NPE 방지
2. Data Class - 보일러플레이트 감소
3. Coroutine - 비동기 처리 (필요시)
4. Java 호환성 - 기존 라이브러리 활용
