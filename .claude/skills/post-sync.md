# Post Sync Skill

프로젝트에서 의미 있는 작업이 완료되면 관련 포스트 업데이트 필요성을 자동으로 확인한다.

## 트리거 조건

다음 상황에서 자동 실행된다.

- git commit/push 완료
- 기능 구현 완료
- 버그 수정 완료
- 설정 변경 완료
- 새 기능 추가

## 점검 절차

### 1단계: 작업 내용 분석

방금 완료된 작업의 핵심 키워드를 추출한다.

- 어떤 기술/도구를 사용했는가?
- 어떤 문제를 해결했는가?
- 새로 추가된 기능은 무엇인가?

### 2단계: 관련 포스트 탐색

`.claude/knowledge/post-index.md`를 참조하여 관련 포스트를 찾는다.

- 태그 기반 매칭
- 제목/요약 키워드 매칭

### 3단계: 업데이트 필요성 판단

| 상황 | 판단 |
|------|------|
| 기존 포스트에서 다룬 기능의 변경/개선 | 업데이트 필요 |
| 기존 포스트의 예제 코드가 변경됨 | 업데이트 필요 |
| 새로운 best practice 발견 | 업데이트 또는 새 포스트 |
| 단순 버그 수정, 리팩토링 | 업데이트 불필요 |
| 완전히 새로운 주제 | 새 포스트 제안 |

### 4단계: 사용자에게 보고

```markdown
## 포스트 동기화 점검

**작업 내용**: {작업 요약}

### 관련 포스트
| 포스트 | 관련도 | 업데이트 필요 |
|--------|--------|--------------|
| 파일명 | 높음/중간/낮음 | 예/아니오 |

### 제안
- 업데이트할 내용 설명
- 또는 "업데이트 불필요" 사유
```

## 예외 상황

다음 경우에는 실행하지 않는다.

- 포스트 자체를 수정하는 작업 (무한 루프 방지)
- 단순 오타 수정
- .gitignore, 설정 파일만 변경
- 테스트 코드만 변경
